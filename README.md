**Customer Service 👥**
Este microservicio se encarga de la gestión integral de los datos maestros de clientes. Es la fuente de verdad para la información personal que consumen otros servicios como el de Cuentas (Account Service).

**🚀 Tecnologías y Stack**
Java 17 (Amazon Corretto)

Spring Boot 3.4.x

Spring WebFlux: Arquitectura reactiva no bloqueante.

Spring Data R2DBC: Persistencia reactiva para PostgreSQL.

PostgreSQL: Base de datos para el almacenamiento de perfiles de clientes.

Lombok: Para la generación automática de boilerplate (Getters, Setters, Builders).

**📊 Diseño de la Base de Datos**
El servicio utiliza PostgreSQL con un esquema sencillo pero eficiente para la identificación y contacto de los clientes.

Definición de la Tabla customers
```sql
CREATE TABLE IF NOT EXISTS customers (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    nombre VARCHAR(100),     -- Nombres del cliente
    apellido VARCHAR(100),   -- Apellidos completos
    email VARCHAR(150),      -- Correo electrónico único
    telefono VARCHAR(20),    -- Número de contacto
    activo BOOLEAN,          -- Estado para manejo de borrado lógico
    created_at TIMESTAMP,    -- Registro de creación
    updated_at TIMESTAMP     -- Registro de actualización
);
```

**🏗️ Funcionalidades Clave**
Gestión de Perfiles: Registro y actualización de datos personales de forma asíncrona.

Validación de Identidad: Asegura que la información básica esté completa antes de permitir la apertura de cuentas en otros servicios.

Alta Disponibilidad: Diseñado para responder de forma rápida a las consultas de orquestación de otros microservicios.

**🔧 Configuración y Puertos**
Puerto del Servicio: 8081

Driver de Base de Datos: R2DBC (Reactive Relational Database Connectivity).

**🛠️ Instalación y Uso**
Clonación:

Bash
git clone https://github.com/IDM-MICROSERVICES/customer-service.git
Compilación:

Bash
mvn clean install
Ejecución:

Bash
mvn spring-boot:run
**🔐 Endpoints Base**
HTTP
GET /customers/listar
Retorna el flujo (Flux) de todos los clientes registrados.

HTTP
GET /customers/{id}
Obtiene el detalle de un cliente específico por su ID.

HTTP
POST /customers/registrar
Registra un nuevo cliente en el sistema.

HTTP
DELETE /customers/{id}
Realiza el borrado lógico o físico del registro del cliente.
